# Архитектура d31

## Границы компонентов

`assistant.py` отвечает за пользовательский сценарий. Он не вызывает `git`
напрямую: клонирование, синхронизация, ветка и commit доступны только через
`GitMCPClient` и локальный stdio-сервер.

`mcp_server.py` владеет каталогом `data/projects`. Идентификатор проекта — первые
16 символов SHA-256 нормализованного Git URL. Сервер принимает только такой
идентификатор, поэтому MCP-вызов не может выбрать произвольный путь на диске.

`rag.py` обнаруживает документацию, создаёт перекрывающиеся чанки с координатами
строк и строит BM25-индекс. Commit и версия алгоритма записываются в metadata;
они являются ключом инвалидации.

`claude_client.py` формирует grounded-запрос: Git URL, branch и commit плюс шесть
лучших фрагментов. Ключ загружается только из `.env` или переменной окружения.

## Последовательность запуска

1. CLI берёт URL из аргумента или последний URL из `data/state.json`.
2. MCP `connect_repository` клонирует или fast-forward обновляет cached clone.
3. MCP `git_branch` и `git_commit` возвращают живой Git-контекст.
4. RAG загружает совместимый индекс либо создаёт новый.
5. Для каждого `/help` выполняются retrieval и отдельный запрос Claude API.

## Отказоустойчивость

- Повреждённый индекс перестраивается.
- Изменение commit или версии индекса инвалидирует кэш.
- Ошибка `git pull` не блокирует уже клонированный проект.
- Ошибка Claude API не завершает CLI-сессию.
- Первая ошибка clone завершает запуск, поскольку локального источника ещё нет.
