# Продукт и текущие ограничения

«Капитолия» — онлайн-настольная экономическая игра. Сервер авторитетен: клиент
отправляет намерения, сервер проверяет правила, рассылает события и актуальный
снапшот. Игровой процесс работает через WebSocket, авторизация и комнаты — через
REST API.

## Авторизация

В текущей реализации токены являются непрозрачными строками без срока действия.
JWT, refresh-токены, OAuth и привязка гостя к внешнему аккаунту описаны в планах,
но ещё не реализованы. Статус пользователя из CRM имеет приоритет над общими
советами: для заблокированного аккаунта повторный вход и переустановка не решат
проблему, тикет нужно эскалировать сотруднику поддержки.

## Комнаты и партии

Комнату создаёт авторизованный пользователь. Остальные сначала выполняют
REST-вход по коду, затем открывают WebSocket. Начать партию может только владелец,
в комнате должно быть минимум два игрока. После старта новые участники не
добавляются.

Активные партии сохраняются в PostgreSQL, если сервер запущен с `DATABASE_URL`.
Без базы состояние живёт только в памяти процесса и не переживает рестарт.

## Версия протокола

Клиент и сервер обмениваются JSON-сообщениями с полем версии `v`. Несовместимые
изменения требуют обновления клиента. При ошибке версии рекомендуйте обновление,
а не сброс профиля.

## Правила ответа поддержки

- Сначала учитывайте факты из тикета и CRM, затем общую документацию.
- Не сообщайте пользователю внутренние идентификаторы и технические поля без
  необходимости.
- Не утверждайте, что проблема исправлена, если нет подтверждения.
- Если документации недостаточно, запросите версию, платформу, код ошибки и шаги
  воспроизведения.
- Заблокированные аккаунты, потерю гостевого профиля с важной партией и вопросы
  персональных данных передавайте человеку.
