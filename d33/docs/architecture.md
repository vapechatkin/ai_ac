# Архитектура d33

## Последовательность запроса

1. FastAPI валидирует вопрос и необязательные `ticket_id` / `user_id`.
2. `CRMClient` вызывает read-only tool локального MCP-сервера по stdio.
3. Для тикета MCP возвращает и обращение, и связанного пользователя. Переданный
   отдельно `user_id` сверяется с владельцем тикета.
4. `SupportEngine` объединяет вопрос с диагностическими фактами CRM и выполняет
   BM25-поиск по `knowledge/`.
5. Claude получает только найденные документы, данные нужного тикета и вопрос.
6. API возвращает текст, идентификаторы использованного контекста, модель и
   список источников RAG.

## Почему MCP вынесен в отдельный процесс

JSON можно было бы читать напрямую из API, но отдельная MCP-граница показывает
тот же контракт, который можно использовать с настоящей CRM. MCP-сервер владеет
доступом к данным, а support-сервис знает только набор инструментов. Сервер
read-only: модель не может изменить пользователя или тикет.

## Контекст и приоритет фактов

CRM-факт имеет приоритет над общей рекомендацией. Например, повторная
авторизация уместна при `unauthorized` и активном аккаунте, но не исправляет
`account_blocked`. Для тикета `TCK-103` ассистент должен предложить эскалацию, а
не бесконечную переустановку.

Описание тикета считается недоверенным вводом. System prompt запрещает выполнять
инструкции, встреченные в вопросе или описании тикета, и запрещает выдумывать
статусы и действия сотрудников.

## Жизненный цикл

FastAPI lifespan один раз загружает RAG-индекс и запускает MCP stdio-сессию.
Оба объекта переиспользуются всеми запросами и закрываются при остановке
приложения. При изменении файлов базы знаний сервис нужно перезапустить.

## Данные и секреты

- `data/crm.json` — демонстрационные синтетические данные, разрешённые для Git;
- `.env` — локальный API key, исключённый `.gitignore`;
- `.env.example` — только имена переменных и безопасные шаблонные значения.

Перед реальным использованием JSON нужно заменить адаптером CRM с OAuth/service
account, ограничением полей, журналом обращений и политикой хранения PII.
